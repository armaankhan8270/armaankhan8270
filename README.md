### 🤖 Claude Code Agentic dbt Pipeline Engine | 2025–2026

**Problem:** Manually scaffolding dbt staging layers, writing schema.yml property files,
and generating boilerplate tests across hundreds of source tables introduces
systematic human error and kills delivery velocity.

**Architecture:**
- Designed a **multi-agent orchestration system** using Claude Code custom skills and
  **Model Context Protocol (MCP) servers** that programmatically connect to live Snowflake
  system catalogs (`INFORMATION_SCHEMA`, `SHOW COLUMNS`)
- Sub-agents read database constraints, infer column semantics, and
  **auto-generate production-ready dbt staging models** with Jinja macros
- A dedicated schema monitoring agent streams Snowflake DDL change events via
  **Snowflake Streams** and triggers automatic model regeneration when upstream schemas drift
- All outputs are validated through a **dbt test runner agent** before PR creation

**Tech Stack:** `Claude Code` · `MCP` · `Snowflake` · `dbt Core` · `Python` · `GitHub Actions`

**Business Impact:**
- ✅ Reduced dbt scaffolding and documentation time by **70%+**
- ✅ Eliminated schema drift failures through autonomous monitoring
- ✅ Cut pipeline onboarding from days to under 2 hours

### ☁️ Apache Airflow Orchestrated Multi-Source Cloud Pipeline | 2025

**Problem:** Processing high-frequency, multi-format payloads from distributed APIs,
flat files, and database CDC streams reliably — with full observability and
automated failure recovery — without constant EC2 compute spend.

**Architecture:**
- Designed **modular Apache Airflow DAGs** using the TaskFlow API with
  custom operators for Snowflake, S3, and external REST source connectors
- Implemented **dynamic DAG generation** — a single factory pattern generates
  source-specific ingestion DAGs from a YAML configuration manifest,
  eliminating repetitive DAG boilerplate entirely
- Built custom **Airflow Sensors** that gate downstream transformation tasks

- Configured **SLA miss callbacks** triggering SNS alerts and automatic
  Snowflake warehouse suspension to control credit spend during anomalies
- Downstream dbt incremental builds trigger on successful Airflow task completion
  via the dbt Cloud API operator

**Tech Stack:** `Apache Airflow` · `AWS S3` · `Lambda` · `EventBridge` ·
`Snowflake` · `dbt Core` · `Great Expectations` · `Docker` · `GitHub Actions`

**Business Impact:**
- ✅ Achieved **100% pipeline observability** with zero silent failures
- ✅ Dynamic DAG factory reduced pipeline onboarding effort by **60%**
- ✅ SLA monitoring + warehouse auto-suspend cut runaway compute costs by **40%**
